Bug vs Feature

Feature

A feature is a small, client-valued function.

Features are well-defined through tickets, wireframes and prototypes

  • Display activities in a carousel
  • Send money to a friend through inline message
  • ...

Bug

A software bug is an error or fault in a computer program that causes it to produce an incorrect result.

Basically, when a feature is not correctly implemented

Not a bug

  • Not breaking specifications
  • Seems illogical but was not spec'ed differently

Cause

Many things can go wrong

You can't guess the cause

(hence, don't try)

Reproducing

The simplest case(s)

"Find me an activity for my 9 years old daughter in Paris that we can do under the rain"

vs

"Activity"

Shortest (but full) path

Always assume a first-time user

  1. Open the bot
  2. Log in
  3. Type "activity"

Reporting

Expected parts

  • Environment — device, operating system, screen size... Anything relevant
  • Step-by-step — path to reproduce
  • Current behavior — what you see (description, highlighted screenshots, ...)
  • Expected behavior — exactly what you expect instead

Anti-patterns

"It doesn't work"

  • A: I'm going to Paris but it doesn't work
  • B: How so?
  • A: Well, I start going but then I'm not there
  • B: What do you mean?
  • A: Going. To Paris. Not working.
  • B: Ok. By car? Plane? Dragon?

"Not as in the design"

  • What isn't?
  • What is wrong with it?
  • How should it be?
  • What is your screen size? Screen density? Device?
